Bash script: Create BLAST database and run BLAST search

BLAST (Basic Local Alignment Search Tool) finds regions of similarity between biological sequences. The program compares nucleotide or protein sequences to sequence databases and calculates the statistical significance.

A BLAST search enables a researcher to compare a subject protein or nucleotide sequence (called a query) with a library or database of sequences, and identify library sequences that resemble the query sequence above a certain threshold. (Wikipedia)

A sequence similarity search is performed with a FASTA file as input. The sequences in this file are compared against a collection of known sequences (another FASTA file), from which a BLAST database is constructed (for higher efficiency of the comparisons). The output is the result of the sequence similarity search. For each input sequence ("query"), it lists all hits, i.e. all significantly similar sequences. The tabular BLAST output format is explained here. The first column contains the query sequence identifier, the second column contains the hit sequence identifier (called "subject"), etc.

Two ways to print a line: performance considerations

Below are two ways of printing the second line of a file (see grymoire.com, tldr.ostera.io).

  • sed -n '2p' dummyfile
  • sed '2q; d' dummyfile

Both work. But which one is preferable, and why? For short files, there is no difference. However, we might need to print the second line of a very large file. Let's perform an experiment. First, generate some random data:

Generate some dummy text files (askubuntu.com: /dev/urandom, Wikipedia: Base64):

base64 /dev/urandom | head -c 1000000 > dummy1.txt
base64 /dev/urandom | head -c 10000000 > dummy2.txt
base64 /dev/urandom | head -c 100000000 > dummy3.txt
base64 /dev/urandom | head -c 1000000000 > dummy4.txt

Now, perform a timing experiment:

for file in dummy*.txt; do
    echo
    echo "--- Test subject: $file ---"
    echo "First approach ("2p") : "; time sed -n '2p' $file > /dev/null
    echo
    echo "Second approach ('2q;d') : "; time sed '2q; d' $file > /dev/null
done

ASCII cat

Python multiline strings start and end with triple quotes (example from this book). Execute Code 1 in the Ipython interpreter:

# Code 1
s = """Dear Alice,

Eve's cat has been arrested for catnapping, cat burglary, and extortion.

Sincerely,
Bob"""

s  # view string representation
print(s)  # print the string

The backslash character \ is a special character, an escape character. It changes the meaning of the subsequent character - either gives it a special meaning, or takes away a special meaning. For example, 'n' is a simple letter, but '\n' is the newline character. This makes writing special characters possible, like:

  • \n newline character (prints as newline)
  • \t tab character
  • \r carriage return character

While the tab character is represented as two characters, \t, it is encoded as one character in ASCII or Unicode (UTF-8) encoding. All Python strings (objects of the str type) are Unicode by default. This allows e.g. to use flowers in Python strings: print(" "), and was one of the main reasons for the creation of Python 3. See how important flowers are? A full list of escape sequences is here.

Prefixing a special character with \ turns it into an ordinary character. This is called escaping. Examples:

  • \' single quote character, e.g. print('It\'s sunny'); a simpler option is print("It's sunny")
  • \" double quote character, e.g. print("\"hello\""); a simpler option is print('"hello"')
  • \\ backslash character, e.g. print('ex\tamp\nle') vs. print('ex\\tamp\\nle')

Another option to take away the special meaning is to use raw strings, e.g. print(r'ex\tamp\nle'). Python docs say: "Both string and bytes literals may optionally be prefixed with a letter r or R; such strings are called raw strings and treat backslashes as literal character."

When you enter a string, Python represents it internally in a "normalized" form. This is why 'foo', "foo", r'foo' and """foo""" produce identical output on the interpreter (try it out). You can construct a string using escape characters, e.g. a = 'abc\ndef'. When you enter a in the interpreter, you see the normalized representation of the string. When you print(a), the backslash characters are interpreted.

FizzBuzz: A simple interview question

The Fizz Buzz game is a popular interview question for beginning programmers. It's said that this question helps filter out a significant percentage of the candidates.

Write a program that prints the numbers from 1 to 100. But for multiples of three, print "Fizz" instead of the number, and for the multiples of five, print "Buzz". For numbers which are multiples of both three and five, print "FizzBuzz". Example:

1, 2, Fizz, 4, Buzz, Fizz, 7, 8, Fizz, Buzz, 11, Fizz, 13, 14, FizzBuzz, 16, ...

(Note that the solution at 6:05 is not necessarily the best in all circumstances. Essentially, you have two conditions, and if both are true, the output corresponds to the combined output of each condition. However, if the requirements change and the output should differ - e.g., you want to output BuzzFizz instead of FizzBuzz, the solution at 5:05 can be preferable, because it has a separate if-clause for this case.)
